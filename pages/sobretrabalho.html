TAGS HTML UTILIZADAS

🏗️ Estrutura do Documento
<!DOCTYPE html> Todas as páginas

Declaração que informa ao navegador que o documento usa HTML5. Deve ser sempre a primeira linha do arquivo. Garante que o navegador renderize a página no modo padrão correto.

Onde foi usado: Primeira linha de todo arquivo HTML do projeto
<html lang="pt-BR"> Todas as páginas

Elemento raiz que envolve todo o conteúdo da página. O atributo lang="pt-BR" indica o idioma principal da página como Português do Brasil — importante para acessibilidade e mecanismos de busca.

Atributo usado: lang="pt-BR"
<head> Todas as páginas

Contém informações sobre a página que não são exibidas diretamente ao usuário: título da aba, links para CSS, metadados de codificação e viewport. É a "cabeça invisível" do documento.

<meta charset="UTF-8"> Todas as páginas

Define a codificação de caracteres como UTF-8, que suporta acentos, cedilha e emojis. Sem isso, caracteres especiais como "ã", "ç" e "é" apareceriam quebrados no navegador.

<meta name="viewport"> Todas as páginas

Controla como a página é exibida em dispositivos móveis. Com width=device-width, initial-scale=1.0, a página se adapta ao tamanho da tela do celular sem zoom automático.

Valor usado: content="width=device-width, initial-scale=1.0"
<title> Todas as páginas

Define o texto exibido na aba do navegador e nos resultados de busca do Google. Cada página tem um título diferente: "Krypton FIT", "Cadastro — Krypton FIT", etc.

<link rel="stylesheet"> Todas as páginas

Conecta um arquivo CSS externo à página HTML. O atributo href indica o caminho do arquivo. Em páginas dentro da pasta pages/, o caminho usa ../ para subir um nível.

Exemplos usados: href="style.css" | href="../cadastro.css"
<body> Todas as páginas

Contém todo o conteúdo visível da página: textos, imagens, formulários, botões. Tudo que o usuário vê na tela está dentro do body.

🧭 Navegação
<nav> Todas as páginas

Tag semântica que representa a barra de navegação principal. Informa aos leitores de tela e mecanismos de busca que aquele bloco contém links de navegação do site.

Classe usada: class="navbar"
<ul> e <li> Todas as páginas

<ul> cria uma lista não ordenada (sem números). <li> é cada item da lista. Usados juntos para criar o menu de navegação com os links Início, Planos, Cadastro e Entrar.

Classe usada: class="nav-links"
<a href> Todas as páginas

Cria links clicáveis. O atributo href define o destino. Usado para links de navegação, botões e o botão do WhatsApp flutuante. Com target="_blank", abre em nova aba.

Exemplos usados: href="../index.html" | href="cadastro.html" | target="_blank"
📝 Conteúdo e Texto
<h1> até <h4> Todas as páginas

Tags de título com hierarquia. h1 é o título principal (mais importante), h2 para seções, h3 para subseções. No projeto, o h1 do hero tem fonte grande com clamp() responsivo.

Exemplo de uso: <h1>FORJE SEU <span class="destaque">MELHOR</span> CORPO</h1>
<p> Todas as páginas

Tag de parágrafo. Usada para textos descritivos como a descrição do hero, textos dos cards de diferenciais e legendas do carrossel.

<span> Todas as páginas

Tag inline para estilizar parte de um texto sem quebrar linha. Usado para colorir palavras em verde como "MELHOR", "FIT" e preços com a classe destaque.

Uso principal: <span class="destaque">MELHOR</span>
<strong> sobretrabalho.html

Deixa o texto em negrito e indica importância semântica. Diferente de apenas usar CSS bold, o strong comunica ao navegador que aquele texto é importante.

<br> index.html

Insere uma quebra de linha dentro de um texto sem iniciar um novo parágrafo. Usado no h1 do hero para separar FORJE / SEU / MELHOR / CORPO em linhas diferentes.

<small> catalogo.html, index.html

Reduz o tamanho do texto. Usado na tabela de planos para deixar o "R$" menor que o número do preço, criando aquele efeito visual de preço destacado.

Uso: <small>R$</small> 89<span>/mês</span>
🏛️ Seções e Layout
<section> index.html, catalogo.html

Tag semântica que agrupa um bloco temático da página. Cada seção tem um propósito: hero, carrossel, diferenciais, tabela de planos. Melhora SEO e acessibilidade.

Exemplos usados: class="hero" | class="secao secao-galeria" | class="secao-tabela"
<header> sobretrabalho.html

Define o cabeçalho da página ou de uma seção. Semânticamente indica que aquele bloco é o topo/introdução do conteúdo.

<main> sobretrabalho.html

Indica o conteúdo principal da página. Deve haver apenas um por página. Ajuda leitores de tela a pular a navegação e ir direto ao conteúdo.

<div> Todas as páginas

Elemento de bloco genérico, sem significado semântico. Usado para agrupar elementos e aplicar estilos. É o elemento de layout mais utilizado no projeto.

Exemplos usados: class="container" | class="hero-conteudo" | class="carousel-wrapper"
🖼️ Mídia
<img> Todas as páginas

Exibe imagens na página. O atributo src indica o caminho da imagem e alt descreve a imagem para acessibilidade e quando a imagem não carrega. Tag vazia, sem fechamento.

Exemplo usado: <img src="imagens/kryptonfit.png" alt="Krypton Fit" class="logo-img">
📊 Tabelas
<table> index.html, catalogo.html

Cria a estrutura de tabela. Usada para a tabela comparativa de planos — exibe quais recursos cada plano inclui de forma organizada em linhas e colunas.

Classe usada: class="tabela-planos"
<thead>, <tbody>, <tfoot> index.html, catalogo.html

thead agrupa o cabeçalho da tabela (nome dos planos e preços). tbody contém as linhas de dados (recursos). tfoot contém o rodapé (botões "Matricule-se").

<tr>, <th>, <td> index.html, catalogo.html

tr é uma linha da tabela. th é uma célula de cabeçalho (negrito por padrão). td é uma célula de dados normal. Juntos formam a grade da tabela.

Classes usadas: class="th-plano destaque-col" | class="td-check td-no"
📝 Formulários
<form> cadastro.html, login.html

Agrupa todos os campos de um formulário. O atributo action define para onde os dados são enviados e method define como (GET ou POST).

Uso no projeto: <form action="#" method="post">
<input> cadastro.html, login.html

Campo de entrada do usuário. O atributo type define o tipo: text (texto), email (valida e-mail), password (oculta senha), date (calendário), tel (telefone), submit (botão enviar).

Tipos usados: type="text" | type="email" | type="password" | type="date" | type="tel" | type="submit"
<label> cadastro.html

Rótulo descritivo associado a um campo de formulário. O atributo for vincula o label ao input pelo ID, melhorando acessibilidade — ao clicar no texto, o campo é focado.

Exemplo usado: <label for="nome">Nome Completo</label>
<select> e <option> cadastro.html

select cria uma lista suspensa (dropdown). option é cada opção da lista. Usado para os campos Sexo e Plano de Interesse no formulário de cadastro.

Opções criadas: Masculino | Feminino | Prefiro não dizer
<button> cadastro.html, index.html

Cria um botão clicável. Com type="submit", envia o formulário. No carrossel, os botões de seta usam onclick para chamar funções JavaScript.

Exemplos usados: type="submit" | onclick="moverCarousel(-1)"
⚙️ Scripts e Estilos Inline
<script> index.html

Incorpora código JavaScript na página. Usado no final do index.html para controlar o carrossel: trocar slides, criar os pontos de navegação, autoplay e suporte a swipe no celular.

Funções criadas: moverCarousel() | irPara() | atualizarCarousel() | resetAutoplay()

PROPRIEDADES CSS UTILIZADAS

🎨 Variáveis CSS (Custom Properties)
:root { --variavel } Todos os CSS

Define variáveis reutilizáveis em todo o CSS. Com :root, ficam disponíveis globalmente. Facilita manter cores e valores consistentes — muda em um lugar, atualiza em todo o projeto.

Variáveis criadas: --verde: #15f145 | --preto: #000 | --branco: #f5f5f5 | --cinza-claro: #888
var(--nome) Todos os CSS

Usa o valor de uma variável CSS. Garante consistência: se mudar --verde, todos os elementos que usam var(--verde) atualizam automaticamente.

Exemplo usado: color: var(--verde); background: var(--preto);
📐 Layout e Posicionamento
display: flex style.css, catalogo.css

Ativa o Flexbox, sistema de layout unidimensional. Usado na navbar para alinhar logo e links lado a lado, nos botões do hero e nos stats. Permite alinhar e distribuir elementos facilmente.

Complementos usados: align-items | justify-content | gap | flex-direction | flex-wrap
display: grid style.css, catalogo.css

Ativa o CSS Grid, sistema de layout bidimensional. Usado nos cards de diferenciais (4 colunas), nos planos (3 colunas) e no FAQ (2 colunas). Ideal para layouts em grade.

Exemplo usado: grid-template-columns: repeat(4, 1fr); gap: 20px;
position: fixed / absolute / relative / sticky Todos os CSS

fixed: fica fixo na tela mesmo rolando (WhatsApp, navbar). absolute: posicionado em relação ao pai (cantos do carrossel, legenda). relative: base para filhos absolute. sticky: fica fixo ao rolar até certa posição (navbar do catálogo).

z-index style.css

Controla a ordem de sobreposição dos elementos. Valores maiores ficam na frente. A navbar tem z-index: 999, o botão WhatsApp tem z-index: 9999 (sempre na frente de tudo).

Valores usados: z-index: 0 | z-index: 1 | z-index: 999 | z-index: 9999
inset: 0 style.css

Atalho moderno para top: 0; right: 0; bottom: 0; left: 0. Usado no overlay do hero (escurecimento da imagem de fundo) e nos anéis do login para que ocupem o espaço inteiro do pai.

overflow: hidden / auto / scroll Todos os CSS

hidden: esconde conteúdo que ultrapassa o elemento (usado no carrossel para esconder slides fora da tela). auto/scroll: adiciona barra de rolagem quando necessário (tabela em telas pequenas).

✨ Visual e Estética
background-image + background-size: cover style.css

Define a imagem de fundo do hero. cover faz a imagem cobrir todo o elemento sem distorcer, cortando as bordas se necessário. background-position: center centraliza o foco da imagem.

Código usado: background-image: url('imagens/fachadaacademia.png'); background-size: cover;
linear-gradient / radial-gradient Todos os CSS

Cria gradientes de cor. linear-gradient: transição em linha reta (usado no overlay do hero — escuro na esquerda, transparente na direita). radial-gradient: transição circular (fundo da página de cadastro).

Exemplo usado: linear-gradient(to right, rgba(0,0,0,0.88) 40%, rgba(0,0,0,0.25) 100%)
box-shadow Todos os CSS

Adiciona sombra a elementos. Usado para criar o efeito de brilho verde nos cards e botões (glow). Pode ter múltiplas sombras separadas por vírgula para efeitos mais ricos.

Exemplo usado: box-shadow: 0 0 30px rgba(21,241,69,0.15);
text-shadow cadastro.css

Adiciona sombra ao texto. Usado no título KRYPTON FIT da página de cadastro para criar o efeito de brilho verde nas letras.

Exemplo usado: text-shadow: 0 0 10px rgba(21, 241, 69, 0.6);
border-radius Todos os CSS

Arredonda os cantos dos elementos. Com valor 50%, cria círculos perfeitos (botão WhatsApp). Com valores assimétricos como 38% 62% 63% 37%, cria as formas orgânicas dos anéis do login.

backdrop-filter: blur() style.css

Aplica desfoque no que está atrás do elemento, criando o efeito de vidro fosco (glassmorphism). Usado no card flutuante do hero para um visual sofisticado sobre a imagem de fundo.

Código usado: backdrop-filter: blur(10px);
filter style.css

Aplica efeitos visuais em elementos. brightness(0) invert(1) torna a imagem branca (ícone do WhatsApp). drop-shadow() adiciona brilho aos ícones de emoji dos diferenciais.

Usos no projeto: filter: brightness(0) invert(1) | filter: drop-shadow(0 0 8px rgba(21,241,69,0.3))
🎬 Animações e Transições
@keyframes style.css, paginalogin.css

Define os passos de uma animação CSS. Cada keyframe define o estado do elemento em determinado ponto (0% = início, 100% = fim). As animações fadeUp e fadeIn fazem os elementos do hero aparecerem suavemente.

Animações criadas: fadeUp | fadeIn | rotate | pulse-verde
animation style.css, paginalogin.css

Aplica uma animação @keyframes a um elemento. Parâmetros: nome, duração, timing, delay, contagem de repetições, direção. Os anéis do login usam infinite para girar sem parar.

Exemplo usado: animation: fadeUp 0.6s ease forwards; animation-delay: 0.3s;
transition Todos os CSS

Cria efeito suave quando uma propriedade muda (ex: no hover). Sem transition, a mudança seria instantânea. Usado em todos os botões, cards e links do projeto.

Valor padrão usado: transition: all 0.3s ease;
transform Todos os CSS

Modifica a aparência sem afetar o layout ao redor. translateY(-2px): sobe o elemento no hover (efeito de levitação dos botões). translateX(): move os slides do carrossel. rotate(): gira os anéis do login. scale(1.1): aumenta o botão WhatsApp.

🔤 Tipografia
@import url(Google Fonts) cadastro.css, catalogo.css

Importa fontes externas do Google Fonts diretamente no CSS. No projeto foram usadas Bebas Neue (títulos impactantes) e Barlow (textos legíveis).

Fontes importadas: Bebas Neue | Barlow | Fira Code
font-size: clamp() style.css

Tamanho de fonte responsivo automático. clamp(mínimo, ideal, máximo) — o navegador escolhe o valor ideal entre os limites. O h1 do hero usa clamp(2.5rem, 8vw, 6rem): no celular fica menor, no desktop maior.

letter-spacing Todos os CSS

Controla o espaçamento entre letras. Valores positivos afastam as letras, criando o visual premium e tecnológico do projeto. Muito usado nos títulos com a fonte Bebas Neue.

Valores usados: 0.05em | 0.1em | 0.15em | 0.3em
text-transform: uppercase cadastro.css

Transforma todo o texto em maiúsculo via CSS, sem alterar o HTML. Usado nos labels dos campos do formulário de cadastro para o visual em caixa alta.

📱 Design Responsivo
@media (max-width: Xpx) Todos os CSS

Media queries adaptam o layout para diferentes tamanhos de tela. Abaixo de 768px (tablet/celular): o menu some, o grid vira 2 colunas, o card do hero é escondido. Abaixo de 480px: tudo em coluna única.

Breakpoints usados: max-width: 900px | max-width: 768px | max-width: 480px
min-height: 100vh style.css, cadastro.css

vh = viewport height (altura da tela visível). 100vh = 100% da altura da tela. O hero e a página de cadastro usam isso para sempre ocupar a tela inteira, independente do tamanho do monitor.
